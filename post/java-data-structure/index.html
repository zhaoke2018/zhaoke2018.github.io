Java StringBuffer 和 StringBuilder 类

String vs StringBuffer vs StringBuilder

String

Create

String s1 = new String();
String s2 = "kezhao";

Get Length

int s2Len = s2.length();

Substring

s2.substring(2, 6); // return "zhao"

Start from 2, end before 6.

StringBuilder and StringBuffer

We need StringBuilder(or StringBuffer) when modify the original String. We know string is immutable in Java, which means when we do String manipulation, it generates a new String and discards the older one for garbage collection.

In single-thread environment or we don’t care about thread safety, use StringBuilder. In other cases, use StringBuffer.

StringBuilder s3 = new StringBuilder(s2.substring(2, 6));

Append

s3.append("ke");

StringBuilder to String

String s2New = s3.toString(); // return "zhaoke"

String to Char Array

char[] s2Char = s2.toCharArray();

Char at Index

char ch = s2.charAt(4); // return 'a'

Find the first char in String

int index = s2.indexOf('e'); // return 1. if not found, return -1
